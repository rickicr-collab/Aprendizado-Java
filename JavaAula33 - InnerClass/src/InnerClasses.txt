CLASSES ANINHADAS INTERNAS:

 - USADA GERALMENTE PARA TRATAMENTO DE INTERFAÇES GRAFICAS E TRATAMENTO DE EVENTOS EM TELA.
 - A CLASSE INTERNA ÉLA É  BEM PARECIDA COM OS MÉTODOS POIS TEM ACESSO A TODOS OS ATRIBUTOS DA SUA 
 CLASSE MAIS EXTERNA INCLUSIVE OS PRIVADOS.
 - AS REGRAS DE METODOS E CLASSES ESTATICAS E NÃO ESTATICAS SERVEM PARA CLASSES INTERNAS TAMBEM.
 - PARA TER ACESSO AOS MÉTODOS DENTRO DA CLASSE INTERNA É NECESSARIO INSTANCIAR UM OBJETO DE SUA CLASSE
 MAIS EXTERNA PARA QUE DEPOIS INSTANCIAR UM NOVO OBJETO DA CLASSE INTRNA QUE É ATRIBUID DA CLASSE EXTERNA.
 - PARA SE UTILIZAR O THIS DENTRO DE UMA CLASSE INTERNA PARA REFERENCIAR UMA CLASSE EXTERNA DEVEMOS CHAMAR A 
 CLASSE EXTERNA DEPOIS UTILIZAR O THIS PARA REFERENCIAR UM ATRIBUTO EXTERNO (EX: CLASSEEXTERNA.THIS.ATRIBUTOEXTERNO).
 
ATRIBUTOS POSSIVEIS NUMA CLASSE INTERNA:
  **DEVIDO A SEREM PARECIDOS OU SEMELHANTES AOS MÉTODOS ATRIBUTOS SEMELHANTES SÃO POSSIVEIS
 - PRIVATE
 - PUBLIC
 - PROTECTED
 - ABSTRACT
 - STATIC
 - STRICTEP ( DECLARAÇÃO QUE PODE SER USADA EM CLASSES INTERNAS)
 
 
 **CLASSES INTERNAS DENTRO DE MÉTODOS**
  - PARA QUE UM MÉTODO DENTRO DE UMA CLASSE INTERNA SEJA EXECUTADO É NECESSARIO CRIAR UM NOVO OBJETO DESTA CLASSE INTERNA
  E QUE ESE MESMO OBJETO CRIADO JA CHAME O MÉTODO INTERNO DA CLASSE INTERNA.
  - CLASSES INTERNAS DENTRO DE MÉTODOS POSSUI AS MESMAS REGRAS QUE OS MÉTODOS:
   * MODIFICADORES POSSIVEIS PARA CLASSES INTERNAS DENTRO DE MÉTODOS SÃO (FINAL , ABSTRACT) NUNCA OS DOIS AO MESMO TEMPO
 
 REGRAS NO JAVA 7:
  - CLASSE INTERNA NÃO PODE USAR ATRIBUTOS INTERNOS LOCAIS NÃO SER QUE SEJAM FINAIS.
  - DEVIDO A CLASSES INTERNAS TEREM UM TEMPO DE VIDA MAIOR QUE O NORMAL DIFERENTE DOS MÉTODOS ONDE SUAS VARIAVEIS
  LOCAIS APOS O MÉTODO FINALIZAR SÃO APAGADAS DA MEMORIA , CLASSES INTERNAS PODEM FICAR ARMAZENADAS E EM ALGUM MOMENTO
  A CLASSE EXTERNA POSSA USAR O ATRIBUTO LOCAL MESMO QUE ELE TENHA SIDO APAGADO DEVIDO A ISSO ATÉ O JAVA 7 ERA NECESSÁRIO 
  ATRIBUIR FINAL PARA OS ATRIBUTOS CRIADOS DENTRO DE MÉTODOS QUE POSSUIAM CLASSES INTERNAS.
  
 REGRAS NO JAVA 8:
 - QUALQUER ATRIBUTO DECLARADO DENTRO DE UM MÉTODO QUE POSSUI CLASSE INTERNA É CONSIDERADO DE FORMA PASSIVA EFETIVAMENTE FINAL
 NO ENTANTO SE O ATRIBUTO NÃO POSSUI O MODIFICADOR FINAL ELE AINDA PODE SER ALTERADO DECLARANDO QUE O ATRIBUTO NÃO É EFETIVO 
 E DEVE SER DECLARADO COMO FINAL.
 
 ** CLASSES ANONIMAS **
  - UMA CLASSE ANONIMA É UMA CLASSE QUE É DEFINIDA EM DETERMINADO PONTO DE CODIGO QUE NÃO POSSUI NOME DEFINIDO E APOS SEU USO 
  ELA PASSA A DEIXAR DE EXISTIR.
  - PARA ORDENAR A LISTA CRIA UMA CLASSE EXTERNA PARA IMPLEMENTAR O METODO COMPARATOR E UTILIZAR A SORT DA BLIBLIOTECA COLLECTIONS
  PARA ORDENAR A LISTA SE NECESSÁRIO.
  - COM AS CLASSES ANONIMAS VOCÊ PODE IMPLEMENTAR O METODO COMPARETO EM UM BLOCO DENTRO DO PARAMETRO NA LINHA DO MÉTODO SORTE
  USANDO CURRELE BRACES E DENTRO DO BLOCO SOBREESCREVER O MÉTODO COMPARE.
  
  ** CLASSES ANINHADAS ESTATICAS **
   - NÃO SÃO DEVIDAMENTE CLASSES PODEM SER CONSIDERADAS MEMBROS ESTÁTICOS.
   - SÓ É POSSIVEL ACESSAR ATRIBUTOS DE UMA CLASSE MAIS EXTERNA DENTRO DE UMA CLASSE MAIS INTERNA SE O ATRIBUTO FOR ESTATICO

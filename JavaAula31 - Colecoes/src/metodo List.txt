COMENTÁRIOS SOBRE COLEÇÕES:

CLASSE LIST

 - LIST QUASE SEMPRE VAI SER USADO COMO VARIAVEL DE REFERENCIA.
 - ATÉ O JAVA VERSÃO 4 VOCÊ NAO TINHA A OPÇÃO DE DETERMINAR OS TIPOS DE VALORES A SEREM INSERIDOS NA LISTA
   ELA ACEITAVA VARIOS VALORES ATÉ MESMO VALORES COMO CLASSES, DEVIDO A ISSO ERA NECESSÁRIO AO REALIZAR
   INTERAÇÃO USANDO FOREACH ESPECIFICAR OBJECT NA VARIAVEL LOCAL DE INTERAÇÃO PARA QUE O JAVA PUDESSE
   ACESSAR A LISTA E IMPRIMIR SEUS VALORES.
 - NA VERSAO 6 VOCE PODERIA ESPECIFICAR O TIPO DE VALOR A SER INSERIDO NA LISTA.
 - NA VERSÃO 7 FOI CRIADO A SINTAXE DIAMOND.
 - QUANDO ESTÁ TRABALHANDO COM ARRAYLIST USANDO FOREACH O MESMO NÃO PERMITE QUALQUER ALTERAÇÃO DIRETA
   NO TAMANHO DO ARRAY DENTRO DO SEU BLOCO DEVIDO AO JAVA JÁ CONHECER OS ASPECTOS DE TAMANHO DO ARRAY
   CASO ISSO OCORRA É GERADA UMA EXCEÇÃO EM TEMPO DE EXECUÇÃO CHAMADA CONCURTENT MODIFICATIONEXCEPTION.
 - NÃO SE PODE CRIAR ARRAYLIST DE TIPOS PRIMITIVOS PORQUE INTERNAMENTE ELES FAZEM CHAMADA PARA VERIFICAR 
   SE SÃO CLASSES WRAPPER SE FOR É CRIADO O ARRAYLIST SE FOR CLASSE PRIMIRITVA É GERADO UM ERRO DE COMPI
   LAÇÃO NO CÓDIGO.
 - USANDO O MÉTODO CLEAR VOCE PODE LIMPAR OS OBJETOS DENTRO DO ARRAY LIST NO ENTANTO AINDA EXISTE A 
   REFERENCIA DO ARRAYLIST POR ISSO NÃO É GERADO O NULLPOINTEXCEPTION.

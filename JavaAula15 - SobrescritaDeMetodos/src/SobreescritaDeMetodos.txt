SOBRE ESCRITA DE MÉTODOS:

 - É UMA FORMA DE SOBREESCREVER MÉTODOS PARA SER UTILIZADOS EM OUTRAS CLASSES DESDE QUE ESTEJAM SEGUINDO A REGRA DA HERANÇA.
 - TODAS AS CLASSES CRIADAS E SUAS SUBCLASSES SÃO FILHAS DA CLASSE OBJECT E PODEM SOBREESCREVER SEUS MÉTODOS.
 - EXISTEM VARIOS MÉTODOS QUE PODEM SER SOBREESCRITOS DA CLASSE OBJECT QUE SÃO UTEIS COMO POR EXEMPLO:
 * MÉTODO TOSTRING(): IMPRIMI OS DADOS DOS ATRIBUTOS EXISTENTES NA CLASSE COMO STRING CONVERTENDO SEUS VALORES.
 * MÉTODO HASHCODE(): 
 * MÉTODO EQUALS(): REALIZA COMPARAÇÕES ENTRE MAIS DE UM VALOR DE ACORDO COM UMA LOGICO PROPOSTA.
 
 OBS: PARA SOBREESCREVER UM MÉTODO TANTO O NOME DO MESMO COMO OS ARGUMENTOS PASSADOS PARA ELE DEVEM SER IGUAIS AOS DO MÉTODO ORIGINAL
 A SER SOBREESCRITO PARA QUE NÃO SEJA GERADO UM ERRO DE COMPILAÇÃO.
 OBS2: O RETORNO DO MÉTODO TO STRING DEVEM SER REDUNDANTE PARA AQUILO QUE DESEJA COMO VALOR DE RETORNO.
 OBS3: O MODIFICAR DE ACESSO NUNCA DEVE SER MAIS RESTRITIVO DO QUE OS ATRIBUTOS DE REFERÊNCIA.

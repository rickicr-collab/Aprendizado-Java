MÉTODOS OU COMPORTAMENTOS:

 - OS MÉTODOS SÃO FUNÇÕES QUE EXEMPLIFICAM OS COMPORTAMENTOS DE UM CLASSE.

**SINTAXE DE UM MÉTODO**
 - PRIMEIRO: MODIFICADOR DE ACESSO (PUBLIC , PACKAGE, PRIVATE, PROTECTED)
 - SEGUNDO: RETORNO DE DADOS QUE O MÉTODO IRA GERÁR EX: (STRING, DOUBLE, VOID(VAZIO) E ETC)
 - TERCEIRO: NOME DO MÉTODO QUE SEGUE A MESMA CONVEÇÃO GLOBAL DAS VARIAVEIS.
 - QUARTO: PARENTESES ONDE SÃO COOCADOS OS PARAMETROS PARA ACÃO INTERNA DOS MÉTODOS.
--------------------------------------------------------------------------------------------------

PARAMETOS:

 - PARAMETOS SÃO AS VARIVAEIS LOCAIS QUE SÃO DECLARADAS DENRO DO PARENTESES E TEM UM ESCOPO LOCAL EFICARAM FUNCIONAIS ENQUANTO O MÉTODO
 ESTÁ ATIVO.
 - OS PARAMETROS ( VARIAVEIS DECLARADAS NOS MÉTODOS) POSSUEM OS MESMOS PRINCIPIOS E REGRAS DAS VARIAVEIS NORMAIS ENTÃO DEVE-SE ATENTAR AS 
 REGRAS AO SE TRABALHAR COM AS VARIAVEIS USADAS COMO PARAMETROS.
 - TODOS OS PARAMETROS PASSADOS DEVEM NO MÉTODOS COMO ARGUMENTOS DEVEM SER PASSADOS AO CHAMAR O MÉTODO CASO CONTRATIO IRÁ GERAR UM ERRO DE 
 COMPILAÇÃO.
 - PARA TIPOS PRIMITIVOS É NECESSARIO TAMBEM SEGUIR AS MESMAS REGRAS AO SE PASSAR ESSES TIPOS DE VARIAVEIS COMO ARGUMENTOS AS REGRAS DE 
 TIPOS PRIMITIVOS TAMBEM SE APLICAM NESSE CASO.
--------------------------------------------------------------------------------------------------- 

RETORNOS DE MÉTODOS:

 - SÃO TIPOS DE VALORES QUE UM MÉTODO PODE RETORNAR AO FINALIZAR SUA EXECUÇÃO. 
 - EXISTEM MUITOS TIPOS DE RETORNO PARA METODOS COMO POR EXEMPLO (TIPO REFERENCE (OBJECT)), LIST, STRING E ETC.
 - É POSSIVEL USAR O RETURN PARA MÉTODOS QUE USAM RETORNO VAZIO COMO UMA FORMA DE QUEBRAR A EXECUÇÃO DO MÉTODO APOS EXECUTAR DETERMINADA AÇÃO.
 
 OBS: VARIAVEIS LOCAIS DENTRO DE UM MÉTODO TEM PRIOIRIDADE DE EXECUÇÃO EM RELAÇÃO A VARIAVEIS GLOBAIS.
 OBS2:  VARIAVEIS PRÉ DECLARADAS DENTRO DE UMA CLASSE NÃO PERDEM SEU VALOR AO SER PASSADAS COMO PARAMETROS PARA MÉTODOS DEVIDO A SER ENVIADAS 
 COMO COPIA E NÃO COMO VALORES PARA SEREM USADAS COMO ARGUMENTOS DEVIDO A ISSO MANTEM SUA INTEGRIDADE E VALOR.
 
 OBS3: QUANDO VOCÊ PASSA OBJETOS DE REFERENCIA COM PARAMETRO EM MÉTODOS , VOCÊ ESTÁ PASSANDO UMA COPIA DOS VALORES DECLARADOS COMO OBJETO 
 ELES SÃO IDENTIFICADOS COMO NUMEROS DE REFERENCIA EM MEMORIA DAQUELE OBJETO NA CLASSE ENTÃO CADA OBJETO TEM SEU PROPRIO NUMERO DE REFERENCIA
 ENTÃO PARA CADA OBJETO QUE VOCÊ CHAMAR O MÉTODO ELE VAI REFERENCIA O OBJETO EM ESPECIFICO CRIADO NA CLASSE.
 
 CUIDADO: QUALQUER ALTERAÇÃO NO MÉTODO IRA ALTERAR OS ATRIBUTOS DA OBJETO DE REFERENCIA DA CLASSE, TANTO DENTRO DA CLASSE ORIGINAL OU EM UMA CLASSE 
 EM QUE VOCE ESTÁ TRABALHANDO.
--------------------------------------------------------------------------------------------------------
 
REFERENCIA COM PALAVRA THIS:

 - QUANDDO VOCÊ ESTA USANDO UM MÉTODO DENTRO DE UMA CLASSE E QUER IMPRIMIR UM ATRIBUTO E SEU VALOR É UTILIZADO COMO PALAVRA RESERVADA PARA ESTA ACÃO 
 A PALAVRA (THIS), QUE VAI REFERENCIAR UM ATRIBUTO EXISTENTE DENTRO DA CLASSE.

UTILIZANDO O VAR ARGS:

 - UM MÉTODO UTILIZADO PARA QUE SE POSSA MINIMIZAR O CODIGO AO SE TRABALHAR COM ARRAYS ONDE O PROPRIO JAVA ELE VAI AUTOMATIZAR A CRIAÇÃO
 DE UM ARRAY AO UTILIZAR O VARARGS COMO PARAMETRO DE UM MÉTODO DESDE QUE AS REGRAS DE TIPOS DE VALORES DE UMA CLASSE POSSAM SER RESPETIDADES 
 ONDE O VARAGRS SEJA O ULTIMO PARAMETRO A SER DECLARADO POIS COMO O FUNÇÃO IRÁ CONVERTER TODOS OS ELEMENTOS COMPATIVEIS COM O ARRAY CRIADO ELE
 NÃO TEM UM LIMITADOR PADRÃO ENTÃO PARAMETROS APOS O VARAGRS SERIAM CONVERTIDOS EM ARRAY , MAIS PARA EVITAR IMCOMPATIBILIDADE OU OUTROS 
 PROBLEMAS O JAVA PROIBI DE OUTROS PARAMETROS VIREM APOS O VARARGS.

 SOBRECARREGANDO SUA OPERAÇÃO EM VEZ DE DIVIDIR ENTRE CLASSES.

  - SERVE PARA PEGAR VALORES DE ATRIBUTOS DO ESCOPO DA CLASSE AO INVES DE PRECISAR DE PARAMETROS ELAS PRECISAM DE RETORNO DO TIPO DE VALOR PARA O ATRIBUTO
